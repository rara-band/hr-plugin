# 개념 매핑 — 사람 회사 ↔ 에이전트 조직

같은 것끼리 나란히. *관념이 안 바뀌는 한 안 바뀌는 토대*만 적는다. (개념은 Claude·Codex 공통.)

| 사람 회사 (HR) | = 에이전트 세계 | 한 줄 메모 |
|---|---|---|
| **직원 (worker)** | **서브에이전트 역할** (`agents/*.md` 한 정의) | *세션이 아님.* 직무기술서 = 프롬프트 + 툴 |
| **팀장 / 관리자** | **오케스트레이터** | 서브에이전트 아님 — 메인 에이전트 |
| 근무 한 번 (교대) | 서브에이전트 **실행 1회** (인스턴스) | 같은 역이 여러 번 일함 |
| 팀 / 부서 | 오케스트레이터 + 서브에이전트들 | = Fleet Loop 트리 |
| 회사 / 조직 | 파이프라인 전체 (= 로스터) | |
| 회사의 방향 | 지금 하려는 일 (목표·태스크) | 거창한 전략 아님 |
| 인건비 | 토큰 | 역별 사용량 |
| 근무 기록 | 로그 / 트랜스크립트 | 누가 언제·뭘·얼마나 했나 |
| 성과 | 산출이 쓰였나 + 품질(eval) | |
| 인사팀 | 이 플러그인 | |
| 인사 담당자 | `hr-analyst` 에이전트 | |

**돌리는 단위:** 팀 하나 = *오케스트레이터 세션 하나*가 그 안에서 서브에이전트를 띄우는 트리. **워커 = 서브에이전트, 관리자 = 오케스트레이터.** 조정은 오케스트레이터를 거친다(서브에이전트끼리 직접 통신 없음). — Anthropic·OpenAI Agents·loop engineering 출처가 다 이 형태.

**에이전트엔 없는 것 (그래서 안 만듦):** 온보딩·램프 · 승진·커리어 · 이직·리텐션 · 9-box(잠재력) · 캘리브레이션 위원회 · 깊은 다층 위계.

**표에 일부러 없는 것:** "유휴냐·과부하냐·너무 비싸냐" 같은 건 *고정 정의가 아니라 매번 판단*이라 뺐다 — 전문가([hr-analyst](../agents/hr-analyst.md))가 그때그때 본다.
